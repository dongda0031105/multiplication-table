系統總覽

核心功能
  • Alpaca API 自動下單（整數股）
  • 每日盈虧報告（JSON Model + HTML View 分離）
  • 每日上午 6:00 Email 通知
  • 有交易即時 Email 推播
  • NASDAQ / NYSE 前十強排名
  • 隔日十檔預測（策略評分）
  • 個股本益比（P/E Ratio）每日計算
  • NAV 線圖 vs NASDAQ / S&P 500
帳戶 & 策略管理
  • 多帳戶支援，同一帳戶同一時間僅用一策略
  • 帳戶可隨時切換策略（JSON only）
  • 新策略只需新增 JSON，不修改 Python 代碼
  • 每月初 + 新資金進入時自動再平衡
  • 每檔持倉上限 10%，只買整數股
  • GitHub Actions 統一走遍所有帳戶執行交易
Dashboard & 報告
  • GitHub Pages 靜態 Dashboard（零費用）
  • 現金水位 / 持倉清單 / 1d・1w・1m 獲利 %
  • 回撤 & NAV 線圖含 NASDAQ / SP500 比對（可勾選）
  • 三大關注類別股票顯示
  • 歷史報告儲存於 GitHub Repo，客戶可回查
  • 報告 Model 以 JSON 儲存，View 為靜態 HTML
⚠️ 投資風險提示：本系統所有排名、績效說明及通知文案，均僅供資訊整理與研究參考，不構成投資建議。股票投資有風險，過去績效不代表未來獲利。請在充分了解個人風險承受能力後謹慎操作。

策略 JSON Schema（交易策略描述規範）

參考 trading_strategy_schema_v2_complete.json ─ 新增策略時只需在 strategies/ 目錄放入新的 JSON 檔，Python 引擎自動載入，不需修改任何 Python 代碼。

頂層欄位 說明 重要子欄位 範例值
strategy 策略基本資訊 id, name, asset_class, market, timeframe, enabled toprank_ma_momentum_v2
universe 股票池篩選條件 exchanges, min_price, min_market_cap, min_avg_volume_20d, exclude_sectors NASDAQ/NYSE，市值 ≥ $2B，20日均量 ≥ 1M
indicators 技術指標定義清單 SMA(20/50/100/200), EMA, RSI, MACD, ADX, ATR, BB, OBV, VWAP type, source, period, outputs
derived_factors 衍生因子（公式計算） ma_gap_20, momentum_60d, volume_ratio_20d, atr_pct (close - sma_50) / sma_50
filters 基本面 & 技術面過濾 fundamental (ROE, 收入增長, D/E), technical (SMA排列, RSI, ADX) AND 邏輯，ROE > 10%，RSI 45–70
ranking 加權百分位排名 factors (11 個), weights, watchlist_top_n=20, buy_top_n=8 momentum_90d weight=0.18（最高）
entry_signals 進場信號（AND 邏輯） rank_in_top_n(8), SMA20穿SMA50, MACD>0, 量比>1.2, RSI<70 五個條件全部成立才進場
exit_signals 出場信號（OR 邏輯） rank落後, SMA死叉, ATR停損(2x), 追蹤停損(10%), 止盈(25%), 跌破SMA200 任一條件成立即出場
portfolio 部位管理規則 equal_weight, max_positions=8, target_pct=10%, cash_buffer=10% 前10檔各10%，整數股，不持分股
risk_management 風控參數 max_daily_loss=3%, max_drawdown=15%, halt_new_entries@10% 超過回撤上限暫停新進場
rebalance 再平衡規則 frequency=weekly(策略內), 系統層每月初 + 新資金事件 不每日再平衡以降低交易頻率
execution 下單執行設定 broker=alpaca, order_type=market, 盤後15分鐘, 財報前後不交易 重試 3 次，退退時間 5 秒
backtest 回測設定 initial_cash=$100k, benchmark=SPY, metrics(11個) CAGR, Sharpe, Max Drawdown 等

  schedule:
    - cron: '30 14 * * 1-5'  # 09:30 ET
  workflow_dispatch:
jobs:
  run-all-accounts:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        account: [acc_001, acc_002]
    steps:
      - uses: actions/checkout@v4

4

自動交易執行 & 再平衡

Alpaca 下單 整數股 再平衡 即時通知
預估工期
2 週
📅 Week 8–9 📋 依賴 Phase 3 通過
📚 交付物
  • ExecutionEngine — 市價單、整數股計算（floor）、重試 3 次
  • 財報前後不交易（earnings_days 緩衝期）
  • Rebalancer — 每月初 + 新資金進入時觸發
  • 倉位差異計算（current vs target）
  • 即時交易通知 Email（下單成功 / 失敗）
  • GitHub Actions trade-execute.yml 完整版
  • 帳戶淨值（NAV）每日計算並存 JSON
🏠 再平衡規則
  • 每月 1 日 09:45 ET 執行（GitHub Actions cron）
  • 新資金進入使現金比例 > 5% 時觸發
  • 不進行每日再平衡（減少交易）
  • 每檔目標 10%，只買整數股（math.floor）
  • 最大換手率 50% / 次（避免過度調整）
  • 扇形最大 30% 行業集中度限制

風險提示 & 限制說明

技術風險
  • Alpaca API 限速（200 req/min）— 需加入 rate limiter
  • yfinance 基本面資料可能延遲或不完整 — 建議備援 FMP API
  • GitHub Actions 每月免費分鐘數限制（公開 repo 無限制，私有 repo 有限額）
  • 市場異常跳空（Gap Down）— 策略 JSON 已含 12% 跳空保護
  • Email SMTP 可能被列為垃圾郵件 — 建議使用 SendGrid 或 SES
投資風險
⚠️ 本系統所有功能僅供資訊整理與研究參考，不構成投資建議。請務必在充分了解個人風險承受能力後謹慎操作。
  • 過去回測績效不代表未來實際獲利
  • 演算法策略在市場劇烈波動時可能失效
  • 本益比（P/E）計算基於歷史 EPS，不反映未來盈利能力
  • 系統僅協助資訊整理與可視化，最終投資決策由使用者負責
  • 建議先用 Paper Trading 至少一個月再切換 Live
